# 📊 CurbOps Dashboard

> **Gridlock 2.0 Hackathon** | Next.js Frontend UI

Welcome to the **CurbOps Dashboard**! This is the frontend interface used by BTP (Bengaluru Traffic Police) officers to instantly identify, visualize, and action the worst traffic hotspots in the city.

This dashboard completely decouples from the heavy data engineering pipeline. It simply fetches the pre-calculated, mathematically sorted `zone_summary.json` and `zones.geojson` artifacts generated by our backend.

---

## 💻 Tech Stack

We completely revamped our initial setup and migrated to a modern **Next.js** architecture for maximum performance and a flawless developer experience.

*   **Framework:** Next.js (React)
*   **Styling:** Tailwind CSS + shadcn/ui for incredibly clean, eye-catching, and accessible UI components.
*   **Mapping:** Leaflet (`react-leaflet`) for visualizing the spatial clustering of traffic violations across Bengaluru.
*   **Charts:** Recharts for dynamic, lightweight drill-down charts (vehicle types & violation types).
*   **Language:** TypeScript for end-to-end type safety.

## 🚀 Getting Started

### Prerequisites
You need [Node.js](https://nodejs.org/) (v18+) and your preferred package manager (npm, yarn, pnpm, or bun).

### Installation & Running Locally

1. **Install dependencies:**
   ```bash
   cd dashboard
   npm install
   ```

2. **Start the development server:**
   ```bash
   npm run dev
   ```

3. **Open the App:**
   Navigate to [http://localhost:3000](http://localhost:3000) in your browser.

## 🎨 UI Features

1. **Hotspot Map:** Visualizes all 2,021 enforcement zones. The size of the circle corresponds directly to the physical radius of the congestion zone.
2. **Priority Drill-down:** Click on any zone to see exactly what vehicles are causing the blockage and the recommended patrol window (e.g., 07:00-08:00).
3. **Action Tiers:** Zones are strictly categorized into **TOW** (Red), **PATROL** (Orange), and **MONITOR** (Blue) based on their Capacity-Blockage Minutes (CBM) score.
4. **Data Quality Toggle:** Users can instantly hide "low confidence" zones (areas with fewer than 5 historical violations) to focus only on guaranteed hotspots.
